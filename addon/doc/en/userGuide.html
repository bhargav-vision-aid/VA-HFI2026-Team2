Remote Element Marker - User Guide

Overview

Remote Element Marker lets you assign a friendly name and an input gesture to a specific UI element so you can return to it later with a single shortcut. This is especially useful when a sighted user is remotely controlling your machine and clicks an element you want to access independently.

Requirements

  • NVDA 2023.1 or later
  • Windows 10 (1809) or later

Default Commands

  • NVDA+Windows+M: Mark element under the mouse.
  • NVDA+Windows+N: Mark element at navigator/virtual caret.
  • NVDA+Windows+Shift+M: Open Marker Manager for the current app.
  • NVDA+Windows+C: Arm capture for the next mouse click (remote click workflow).
  • NVDA+Alt+L: Open the marker list for quick activation.

How To Use

1) Mark an Element Under the Mouse

  1. Move the mouse over the target element.
  2. Press NVDA+Windows+M.
  3. Enter a friendly name.
  4. Enter a gesture identifier (or use Capture) and press OK.

2) Mark an Element at the Navigator

  1. Move the navigator object (or virtual caret in browse mode) to the target element.
  2. Press NVDA+Windows+N.
  3. Enter a friendly name and gesture, then press OK.

3) Remote Click Workflow (Sighted User)

  1. Press NVDA+Windows+C to arm capture.
  2. The sighted/remote user clicks the element.
  3. The marker dialog opens. Enter a friendly name and gesture, then press OK.
Press NVDA+Windows+C again to cancel capture if needed.

4) Capture a Gesture

  1. In the marker dialog, select “Capture Gesture”.
  2. Perform the gesture you want to use (keyboard, braille, touch, etc.).
  3. If multiple identifiers are available, choose the desired one.

The captured value is stored as a normalized NVDA gesture identifier, for example kb:NVDA+shift+v or kb(laptop):NVDA+t.

Gesture assignment is optional. You can save a marker without a shortcut and activate it from the marker list.

5) Activate a Marker

  1. Focus the same application (and document if applicable).
  2. Press the assigned gesture.
  3. NVDA will attempt to locate the element, move to it, and activate it.

Or press NVDA+Alt+L to open the marker list and activate a marker by name.

6) Manage or Delete Markers

  1. Press NVDA+Windows+Shift+M.
  2. Select a marker and press “Delete Selected” to remove it.

Input Gestures Dialog Integration

All marker gestures appear under the category Remote Element Marker in NVDA’s Input Gestures dialog. Each marker is listed as a script with its friendly name. You can view or change the gesture there if you prefer.

Settings

Open NVDA Settings and select Remote Element Marker.

  • Announce custom labels when navigating elements: When enabled, NVDA will announce your custom label when you navigate to a marked element (focus, object navigation, etc.).
Disclaimer: This is best-effort matching. If the UI changes, the label may not be applied or could apply to a different element. Disable this option if you prefer NVDA’s default labels.

Storage and Scope

  • Markers are stored in: %appdata%\nvda\remoteElementMarkers.json.
  • Markers are scoped to the application and (for web documents) the document identifier.

Known Limitations

  • Element matching uses a best-effort search and may fail if the UI changes.
  • Web markers are scoped to a specific document identifier and may not carry across pages.
  • Some elements may not support default action; NVDA will still attempt to focus them.

Troubleshooting

  • “Gesture already assigned”: Choose a different gesture or remove the existing binding.
  • “Element not found”: Re-mark the element; the UI may have changed.
  • No markers in manager: Ensure you are in the same application context.
