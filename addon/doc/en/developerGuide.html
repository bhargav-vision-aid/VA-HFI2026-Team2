Remote Element Marker - Developer Guide

Purpose

This add-on lets users create markers for UI elements, assigning a friendly label and a gesture to quickly return to and activate the element. It supports a remote-click workflow, gesture capture via NVDA’s inputCore, and optional label announcement when navigating.

Project Structure

  • addon/globalPlugins/remoteElementMarker/__init__.py: Main plugin entry point and orchestration.
  • addon/globalPlugins/remoteElementMarker/gui.py: Marker dialogs and gesture capture UI.
  • addon/globalPlugins/remoteElementMarker/storage.py: Marker persistence and app scoping.
  • addon/globalPlugins/remoteElementMarker/signature.py: Element signature generation.
  • addon/globalPlugins/remoteElementMarker/resolver.py: Element resolution logic.
  • addon/globalPlugins/remoteElementMarker/bindings.py: Gesture normalization and validation.
  • addon/globalPlugins/remoteElementMarker/settings.py: Settings panel for optional label announcement.
  • addon/doc/en/userGuide.html: End-user documentation.
  • addon/doc/en/developerGuide.html: This file.

Core Flow

  1. User captures an element (mouse, navigator, or remote click).
  2. Signature is generated and stored in JSON by app scope.
  3. User assigns a gesture, which is normalized and bound dynamically.
  4. When the gesture is invoked, the element is resolved and activated.

Key Components

Global Plugin (__init__.py)

  • GlobalPlugin: Orchestrates markers, gesture bindings, and settings.
  • scriptCategory: "Remote Element Marker" (controls Input Gestures category).
  • Default scripts:
    • script_markElementFromMouse
    • script_markElementFromNavigator
    • script_openMarkerManager
    • script_openMarkerList (gesture: NVDA+Alt+L)
    • script_armClickCapture
  • Dynamic script registration: Each marker creates a script_invokeMarker_* on the class so it appears in NVDA Input Gestures.
  • Overlay class: FriendlyNameOverlay optionally overrides name to announce custom labels when navigating.

Storage (storage.py)

  • MarkerStore: Loads/saves marker JSON in user config.
  • get_app_key: Builds app scope key using process + module + window class + document identifier.
  • get_document_identifier: Uses treeInterceptor document identifier when available (web).

Signature (signature.py)

  • generate_signature: Builds a deterministic signature for UIA, Browse Mode, or IAccessible objects.
  • Hashing: Uses MD5 of primary signature JSON for marker identity.

Resolver (resolver.py)

  • resolve_element: Naive tree traversal matching based on primary signature.
  • Note: Fast-path UIA lookup is not implemented yet (future improvement).

Bindings (bindings.py)

  • normalize_shortcut: Uses inputCore.normalizeGestureIdentifier if available.
  • Falls back to minimal normalization when inputCore is missing.

GUI (gui.py)

  • MarkerDialog: Collects friendly name and gesture.
  • Gesture capture: Uses inputCore.manager._captureFunc (NVDA standard).
  • MarkerManagerDialog: Lists and deletes markers for current app.

Settings (settings.py)

  • RemoteElementMarkerSettingsPanel: Checkbox to enable/disable label announcement.
  • Writes to config.conf["remoteElementMarker"]["announceLabels"].

Configuration

  • Markers stored at: %appdata%\\nvda\\remoteElementMarkers.json
  • Settings stored in NVDA config: [remoteElementMarker] section

Debugging Tips

  • Enable NVDA debug logging for input issues.
  • Check marker JSON for signature correctness.
  • Use Input Gestures dialog to verify marker scripts/gestures.

Potential Improvements

  • Implement UIA fast-path lookup with IUIAutomation.
  • Add fuzzy matching for resilient element resolution.
  • Support per-marker scope configuration (window/class/document).
  • Add edit/rename functionality in MarkerManagerDialog.
This guide is designed to speed up onboarding and debugging. For NVDA API details, refer to the NVDA Developer Guide and local NVDA source.
